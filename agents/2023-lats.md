# Language Agent Tree Search (LATS): Unifying Reasoning, Acting and Planning in Language Models

- **作者**：Andy Zhou, Kai Yan, Michal Shlapentokh-Rothman, Haohan Wang, Yu-Xiong Wang
- **机构**：University of Illinois Urbana-Champaign (UIUC)
- **发表**：ICML 2024 · arXiv:2310.04406（2023-10）
- **链接**：[arXiv](https://arxiv.org/abs/2310.04406)
- **主题标签**：`#agent` `#lats` `#mcts` `#search` `#planning` `#reasoning` `#tool-use` `#self-reflection`

## 一句话总结

> 把 **MCTS（蒙特卡洛树搜索）** 引入 language agent，统一**推理、行动、规划**三种能力：LLM 同时充当行动者与价值评估器，结合环境反馈与自我反思，在动作/思维树上做探索-利用平衡的搜索——无需梯度训练即显著提升 agent 决策质量。

## 背景与动机

- ReAct 是顺序、贪心的，缺乏前瞻与回溯；ToT 有搜索但停留在纯推理、不与外部环境交互、也无 MCTS 的统计回传。
- 目标：第一个把 LLM 的**推理（reasoning）+ 行动（acting）+ 规划（planning）** 统一起来的通用框架——既能调工具/与环境交互，又能像 AlphaGo 那样用 MCTS 深思熟虑。

## 核心方法

- **MCTS over language agent**：在「选择 → 扩展 → 模拟/评估 → 回传」的 MCTS 框架里展开节点（每个节点是一个状态/行动）。
- **LLM 多重角色**：同一 LLM 既作 **agent（生成行动）** 又作 **value function（评估节点价值）**。
- **Self-Reflection 自反思**：失败时生成语言反思（借鉴 Reflexion），用作额外信号引导探索。
- **环境接地（external feedback）**：把真实环境/工具的观察纳入评估，使搜索 grounded 而非空想。

## 实验与结论

- **HumanEval（编程）**：GPT-4 pass@1 达 **92.7%**。
- **WebShop（网页导航）**：GPT-3.5 平均分 **75.9**，可与基于梯度微调的方法相媲美。
- 也在交互式 QA、数学推理上评测，均有提升；全程无需梯度优化。

## 个人评注

- 启发：把 [[2012-mcts-survey]] 的经典 MCTS 与 LLM agent 缝合，是「test-time search」在 agent 上的集大成者——ToT（树搜索）+ ReAct（行动）+ Reflexion（反思）的统一体。
- 关联 [[2023-tree-of-thoughts]]：ToT 用 BFS/DFS，LATS 升级为带探索-利用与价值回传的 MCTS，并加入真实环境交互。
- 关联 [[2022-react]]、[[2023-reflexion]]：分别贡献了「行动空间」与「语言反思」；LATS 把二者纳入搜索框架。
- 关联 [[2019-the-bitter-lesson]]：以「搜索 + 学习到的价值评估」换性能，是 Bitter Lesson 在 LLM agent 上的体现。
- 局限：MCTS 需大量 LLM 调用，延迟/成本高；价值评估依赖 LLM 自评的准确度；可并行性受树结构限制。
